  What changed:
- Added a production-safe fallback in `src/app/api/monitoring/launch-status/route.ts` so missing runtime smoke artifacts fall back to persisted synthetic monitor rows instead of returning a 500.
- Added route coverage for the fallback path and refreshed the tracked launch smoke artifact after rerunning the smoke suite.

Why:

- The production Vercel deployment does not expose `.artifacts/launch-smoke-report.json` to the serverless runtime, so the route needed a resilient fallback to preserve launch observability.
- Fresh persisted monitor rows were needed so the fallback reports the current deployed state instead of stale historical failures.
