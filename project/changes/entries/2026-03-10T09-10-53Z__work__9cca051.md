# Project Change Entry

- Date/time (UTC): 2026-03-10T09:10:53Z

What changed:

- Added a new database migration to enable RLS on canonical proof/verification tables.
- Added owner-scoped `FOR ALL` policies for `proof_artifacts`, `proof_packs`, `proof_pack_items`, and `verification_records`.

Why:

- Canonical tables were created without explicit RLS policies, creating cross-tenant data exposure risk via Supabase client roles.

How to verify:

- Run `npm run lint`.
- Run `npm run typecheck`.
- Run `npm run docs:freshness`.
- Apply migrations and confirm `pg_tables.rowsecurity = true` and expected policies exist in `pg_policies` for all four tables.

Open risks / TODO:
