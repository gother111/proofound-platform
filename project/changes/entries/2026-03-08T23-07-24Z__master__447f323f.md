  What changed:
- Added canonical match score storage fields on `matches` and a migration that backfills legacy rows, marks them stale, and installs DB triggers to mark matches stale or hide them when score inputs and matching consent change.

- Updated shortlist, feed, and explain APIs to rank and explain from canonical score artifacts and to suppress stale exact-rank behavior.
- Added focused contract and review-contract tests for determinism, null handling, tie-breaking, and stale-rank suppression.

Why:

- Stored matches need one reproducible scoring artifact so fairness review, explanation rendering, debugging, and recompute behavior all point at the same persisted contract.
- The previous implementation mixed ad hoc weighting, semantic boosts, and `vector`-local explanation logic across multiple writers, which made replay and audit unsafe.
