What changed:

- Added resend support for sent verification requests via `POST /api/expertise/verifications/sent/[requestType]/[requestId]` with status gating for skill, impact story, and bundled custom-request flows.
- Added resend UI actions on the Individual Verifications page (`Resend request` / `Resend bundle`) with loading, toast handling, and refresh behavior when new records are created.
- Expanded custom verifier relationship support from 3 values to an 8-option selectable set with backward compatibility for legacy `external`.
- Added relationship mapping helper from custom relationships to skill verifier source (`peer | manager | external`) and applied it to linked skill request creation and bundle resend cloning.
- Added migration `20260228130000_expand_custom_verifier_relationship_options.sql` and updated Drizzle schema enum to align with database constraint.
- Updated API/UI tests for resend flow and expanded relationship behavior.

Why:

- PRO-103 requires users to resend verification requests directly from the Verifications page and choose from more verifier relationship options when sending custom verification bundles.
