What changed:

- Canonicalized live org team role output to `org_owner`, `org_manager`, and `org_reviewer` across the organization team API, shared dashboard data fetcher, and `TeamRolesCard` UI contract.
- Added `src/lib/organizations/team.ts` as the shared canonical team-data assembler, including legacy input normalization, canonical role ordering, and canonical `stats.byRole` counts.
- Restricted legacy role names to explicit compatibility normalization in `src/lib/authz/membership.ts` and renamed schema constants in `src/db/schema.ts` so canonical and legacy values are clearly separated.
- Aligned org-role checks in runtime helpers and mocks, added a forward SQL migration `20260318180500_align_canonical_org_role_policies.sql`, and updated `src/db/policies.sql` so membership and invite policy follows owner/manager/reviewer least privilege.
- Added focused tests for membership normalization, the organization team route, and extended RLS separation expectations for owner, manager, and reviewer behavior.

Why:

- Locked MVP and authz policy define canonical org-role truth as `org_owner`, `org_manager`, and `org_reviewer`, but runtime outputs and database policy still leaked legacy `owner/admin/member/viewer` semantics.
- Team membership and invitation mutation needed to stay owner-only to preserve least privilege and prevent privacy-gate drift.
