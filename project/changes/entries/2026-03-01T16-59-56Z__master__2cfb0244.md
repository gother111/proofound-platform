# Project Change Entry

- Date/time (UTC): 2026-03-01T16:59:56Z
- Branch: master
- Base commit: 2cfb0244
  What changed:
- Updated `src/middleware.ts` to stop middleware-level CSRF cookie minting for `/api/csrf-token`, keeping that endpoint as the single token issuer.
- Hardened `src/lib/api/fetch.ts` with origin-scoped CSRF cache, fail-fast behavior when token fetch fails, and one-time retry when a CSRF 403 mismatch is detected.
- Extended `src/lib/__tests__/api-fetch.test.ts` with regressions for token-fetch failure and CSRF retry behavior.
- Added `src/lib/__tests__/middleware-csrf.test.ts` to verify `/api/csrf-token` middleware pass-through and regular API CSRF enforcement.

Why:

- Users still hit `CSRF validation failed` on CV wizard analyze uploads.
- The previous flow allowed mutating requests to proceed without a valid CSRF token and had duplicate token issuance on the token endpoint path.

How to verify:

- `npm run test -- src/lib/__tests__/api-fetch.test.ts src/lib/__tests__/csrf.test.ts src/lib/__tests__/middleware-csrf.test.ts`
- `npm run test -- tests/ui/cv-import-wizard.test.tsx tests/ui/cvjd-auto-suggest.test.tsx`
- `npm run typecheck`
- `npm run lint`
- `npm run build`
