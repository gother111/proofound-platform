# Project Change Entry

- Date/time (UTC): 2026-02-26T08:00:43Z
- Branch: master
- Base commit: 39f3994b

What changed:

- Added `requireApiAuthContext()` in `src/lib/auth.ts` to provide a single session-aware Supabase client (`allowCookieWrite: true`) plus authenticated user for API handlers.
- Refactored API routes under `src/app/api/**/route.ts` to use API auth context and return `401` JSON for unauthenticated requests.
- Fixed PRO-74 hot path in `src/app/api/expertise/user-skills/[id]/verification-request/route.ts` by removing split auth/client creation and using one auth context for read and insert.
- Updated API tests to mock `requireApiAuthContext` and validate `401` JSON behavior for unauthenticated requests.

Why:

- Prevent auth/session drift in API routes where read and write operations previously used different auth client contexts.
- Standardize API auth failure contract to explicit machine-readable `401` responses.
- Remove known failure mode behind `Failed to create verification request` in PRO-74.
