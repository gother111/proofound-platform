# Project Change Entry

- Date/time (UTC): 2026-03-10T16:31:26Z
- Branch: master

  What changed:
- Reduced org launch navigation and route entry points to the MVP corridor: Overview, Assignments & Matches, Trust Profile, and Public Trust Profile.
- Gated non-MVP org routes like candidates, shortlist, projects, team, members, settings, integrations, goals, and opportunities behind explicit launch notices or redirects.
- Reworked org trust-profile surfaces so the writable org API only accepts launch trust fields, added a lean trust-profile editor, and aligned work-culture updates with canonical org roles.
- Hardened assignment publish and draft persistence by reusing shared publish validation, returning structured publish block reasons, and allowing explicit org slug context for draft create/update flows.
- Added targeted tests for publish block reasons, basic/advanced publish rules, reviewer restrictions, and the reduced org navigation/settings corridor.
- Finished repo-wide verification cleanup by updating the remaining stale proof-pack and duplicate-request mocks in `tests/actions/profile.test.ts` and `tests/lib/public-portfolio-projection.test.ts`.

Why:

- The canonical PRD and implementation audit both require a much narrower org launch story than the older broad org SaaS shell.
- Assignment publishing needed deterministic validation feedback instead of generic or silent failures, especially inside the basic-mode launch corridor.
- Reviewer access needed to remain review-scoped while owner and manager mutation paths stayed limited to launch-safe trust-profile fields.
