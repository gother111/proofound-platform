# Project Change Entry

- Date/time (UTC): 2026-03-02T09:27:46Z
- Branch: master
- Base commit: fa220c66
  What changed:
- Flattened Gemini structured output schema from nested `documents[].skills[]` to top-level `skills[]` to avoid Gemini schema depth/state failures.
- Added robust Gemini JSON parsing fallback in client (`parseJsonLenient`) for fenced/embedded JSON responses.
- Added invalid-JSON recovery in extractor by retrying same model with boosted output token cap before fallback-model retry.
- Added taxonomy shortlist performance guards: reduced default seed breadth, bounded concurrency, per-seed timeout, and per-document timeout budget.
- Removed duplicate deterministic pass in wizard route on Gemini failure and reused baseline extraction payload.
- Made wizard client recovery unconditional for timeout and multipart metadata invalid failures (removed `NEXT_PUBLIC_CV_IMPORT_CLIENT_FALLBACK_ENABLED` gating).
- Added additive metadata fields (`ai_schema_mode`, `timings`) to CV suggest/wizard metadata schemas and wizard route responses.
- Updated env/docs defaults and variables for shortlist performance settings.
- Added/updated tests for new wizard fallback behavior and Gemini config defaults.

Why:

- Live Gemini calls were failing with nested structured output constraints and JSON truncation, causing non-recoverable wizard failures.
- Shortlist preselection latency could consume most of the route timeout budget.
- Client-side fallback needed to be always available for end-to-end reliability independent of env misconfiguration.
