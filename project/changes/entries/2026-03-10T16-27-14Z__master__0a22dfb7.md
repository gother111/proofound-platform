# Project Change Entry

- Date/time (UTC): 2026-03-10T16:27:14Z
- Branch: master
- Base commit: 0a22dfb7
  What changed:
- Extended `proof_packs` into a first-class canonical proof object with subject, lifecycle, freshness, verification, and portability fields.
- Added `src/db/migrations/20260310190000_canonicalize_proof_pack_core.sql` to backfill compatibility packs for orphaned legacy artifacts and initialize pack rollups.
- Added `src/lib/proofs/canonical-pack.ts` as the shared aggregate/projection layer for owner-full and public-safe Proof Pack reads.
- Switched proof-trust snapshots, profile completion, readiness, matching interest overlap, portfolio export data, public portfolio rendering, and user export/import onto canonical pack state instead of `skill_proofs` counts.
- Added proof-first tests for canonical pack projection/privacy and data portability import scope handling.

Why:

- The PRD requires Proof Pack to be the canonical reviewable proof object and Artifact to be the atomic evidence unit.
- Trust, matching, public portfolio, and portability were still reading legacy loose-evidence tables, which caused competing semantics and weak privacy boundaries.
- Canonical owner-full and public-safe projections are now distinct so hidden child artifacts cannot leak through export or public portfolio views.
