  What changed:
- Updated `src/lib/auth.ts` so `resolveUserHomePath()` fetches `handle` and routes incomplete individual accounts to `/onboarding` instead of `/app/i/home`.

Why:

- Production login for an individual account without a handle was authenticating successfully but routing into the individual shell too early, which surfaced the generic post-login error state.

How to verify:

- Run `npm run lint`.
- Run `npm run typecheck`.
- Run `npm run test`.
- Run `npm run test -- tests/lib/auth-request-cache.test.ts tests/api/auth-callback-route.test.ts`.
- Run `npm run test:privacy:extended`.
- Run `npm run build`.
- Manually sign in with an individual account missing a handle and confirm the redirect goes to `/onboarding`.
