# Project Change Entry

  What changed:
- Updated the persisted launch monitor fallback to keep only rows whose `monitor_key` exists in the current `LAUNCH_MONITOR_DEFINITIONS`.
- Added a regression test covering retired persisted monitor keys so the launch status route cannot regress to mixed historical monitor output.

Why:

- Production Vercel runtime could not read `.artifacts/launch-smoke-report.json`, so the route fell back to DB-backed persisted rows.
- The persisted dataset still contained older monitor contracts, which incorrectly marked launch status unhealthy even though the current launch monitor run was green.

How to verify:

- `npx vitest run src/lib/launch/__tests__/synthetic-monitors.test.ts src/app/api/monitoring/__tests__/launch-status-route.test.ts`
- `npm run lint`
- `npm run build`
- After deploy, confirm `curl -sS https://proofound.io/api/monitoring/launch-status` reports only the active monitor set and no stale historical failures.

Open risks / TODO:
