  What changed:
- `src/components/ui/transition-provider.tsx`
  - Disabled first-mount opacity gating with `AnimatePresence initial={false}` and `motion.div initial={false}` to prevent hidden first paint on client route transitions.

  - Removed top-level `motion.div` wrappers with opacity-based initial states from section blocks.
- `src/hooks/useProfileData.ts`
  - Added explicit profile-load lifecycle state: `loadError`, bounded retry (`2` attempts), and `retryLoad`.
  - Converted redirect/not-found digest handling to controlled navigation (`window.location.assign` / reload) instead of throwing from the effect chain.
- `src/components/profile/EditableProfileView.tsx`
  - Added deterministic non-blank error fallback card with retry and dashboard actions.
  - Added `data-testid="individual-profile-root"` and `data-testid="profile-load-error"`.

- `e2e/profile/profile-nav-no-blank.spec.ts`
  - Added regression coverage for sidebar navigation to individual and org profile routes with explicit "no blank screen" assertions.

Why:

- Client-side navigation to profile routes could intermittently render as blank/white until hard refresh due to overlapping opacity-based route and section transitions plus transient initial fetch errors.
- This patch prioritizes first-paint reliability and explicit failure states over fragile entrance opacity effects.
