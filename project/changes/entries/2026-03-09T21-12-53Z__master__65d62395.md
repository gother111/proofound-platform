# Project Change Entry

What changed:

- Updated `PRD_TECHNICAL_REQUIREMENTS.md` with a decision-complete MVP public portfolio policy that makes the stable portfolio a day-1 output while keeping indexing opt-in, snippets non-canonical, and reveal-mode separate from public-mode.
- Tightened stable public portfolio rendering so handle or slug is the public fallback identity, public pages respect public-only field visibility for display name and skills, and reveal-gated proof links do not render on the stable public route.
- Made public snippets and embeds explicitly noindex and non-canonical, and added privacy-safe analytics for public snippet views and unavailable snippet requests.
- Added `public_portfolio_viewed` analytics on stable public portfolio pages for both individual and organization routes, and prevented historical handle or slug redirects from forwarding into unpublished public pages.

Why:

- The previous implementation already had the right launch state machine for public portfolios, but the operational PRD policy was too thin and token snippets still behaved too much like canonical public pages.
- The approved MVP policy requires day-1 live sharing without weakening privacy-first promises, which meant aligning metadata, view tracking, and field-visibility enforcement with the stable public portfolio contract.
