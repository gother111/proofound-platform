# Project Change Entry

- Date/time (UTC): 2026-03-01T22:50:53Z
- Branch: master
- Base commit: 83bc866c

What changed:

- Updated `next.config.js` build toggles to auto-detect Vercel builds (`VERCEL` or `VERCEL_ENV`) and enable:
  - `eslint.ignoreDuringBuilds`
  - `typescript.ignoreBuildErrors`
  - `experimental.webpackBuildWorker`
  - `experimental.webpackMemoryOptimizations`
- Replaced `prepare` script in `package.json` from direct `husky install` to `node ./scripts/prepare-husky.mjs`.
- Added `scripts/prepare-husky.mjs` to skip Husky installation when `.git` is unavailable (Vercel build context), avoiding fatal install warnings.

Why:

- Vercel deployments were being OOM-killed during `next build` (`SIGKILL`) in an 8 GB container.
- Existing config only skipped lint/type checks when `NEXT_SKIP_BUILD_VALIDATION=1` was manually set, so Vercel did not get the mitigation automatically.
- Vercel install phase logged `fatal: not in a git directory` from Husky because `.git` is excluded in build context.

How to verify:

- `export NVM_DIR="$HOME/.nvm" && . "$NVM_DIR/nvm.sh" && nvm use 20.20.0`
- `VERCEL=1 VERCEL_ENV=preview npm run build`
  - Expect output to include:
    - `✓ webpackBuildWorker`
    - `✓ webpackMemoryOptimizations`
    - `Skipping validation of types`
    - `Skipping linting`
- `npm run lint` (passes with 2 existing `@next/next/no-img-element` warnings in landing files)
- `npm run typecheck`

Open risks / TODO:

- Vercel builds now rely on CI lint/typecheck as the source of truth for static validation.
- If branch protection or required checks are relaxed, lint/type regressions could ship.
