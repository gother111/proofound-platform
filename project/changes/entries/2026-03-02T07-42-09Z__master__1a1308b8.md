  - `src/components/expertise/cv-import/CvImportWizard.tsx`
    using existing local text extraction and JSON retry path.
- Updated tests to verify new behavior and code propagation:
  - `tests/lib/python-cv-proxy.test.ts`
  - `tests/api/cv-import-suggest-route.test.ts`
  - `tests/api/cv-import-wizard-suggest-route.test.ts`
  - `tests/ui/cv-import-wizard.test.tsx`
  - `tests/python/test_cv_import_multipart_errors.py`

Why:

- Upload analysis was blocked by a UTF-8 metadata error even for valid files because multipart boundary forwarding was missing and client fallback did not trigger for this specific failure class.
- The fix removes the transport cause and adds resilient auto-recovery so users are not blocked.
