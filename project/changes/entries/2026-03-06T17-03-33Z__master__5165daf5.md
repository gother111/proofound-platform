# Project Change Entry

- Date/time (UTC): 2026-03-06T17:03:33Z
- Branch: master
- Base commit: 5165daf5
  What changed:
- Restored legacy individual login routing in `src/lib/auth.ts` so `resolveUserHomePath()` sends any `persona === 'individual'` user to `/app/i/home` without treating `handle` as an auth gate.
- Updated `src/app/onboarding/page.tsx` to use `getIndividualProfileCompletionState(user.id).isCoreProfileComplete` for individual onboarding compatibility instead of treating a missing `handle` as incomplete onboarding.
- Added a new route-level regression in `tests/routes/onboarding-page.test.ts` and updated `tests/lib/auth-request-cache.test.ts` to lock the legacy no-handle behavior.

Why:

- A production regression was introduced by using `handle` as an onboarding and shell-access requirement even though legacy individual accounts could complete onboarding before handles existed.
- The intended behavior is dashboard access after login for legacy individual users, while keeping handle checks only on routes that actually need a public portfolio URL.

Open risks / TODO:
