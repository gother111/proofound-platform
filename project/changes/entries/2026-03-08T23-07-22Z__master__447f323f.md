What changed:

- Added a canonical authz matrix under `src/lib/authz/` with explicit resource and action enums, role semantics, reveal scope helpers, export helpers, audit visibility helpers, and a shared `authorize(...)` resolver.
- Rewired org auth surfaces to use the canonical policy for org profile updates, invites and membership management, assignment org-context resolution, shortlist and review projection filtering, dashboard reads, and org export routes.
- Added explicit org audit export and platform break-glass org audit read routes, plus a reusable break-glass helper that requires a reason and writes to `admin_audit_log`.
- Tightened org audit log RLS in `src/db/policies.sql` and added `src/db/migrations/20260309013000_block11_tighten_org_audit_log_policy.sql` so only org `owner` and `admin` can read org audit logs.
- Added Block 11 regression tests for the matrix, review projection behavior, org export restrictions, and org audit export and break-glass flows.

Why:

- Block 11 requires one explicit, enforceable permission model for Proofound MVP instead of thin role-name checks and implicit org membership semantics.
- Candidate reveal, export, and audit visibility are privacy-sensitive capabilities and must not widen accidentally through generic review or membership access.
- The previous org audit log policy was too broad because any active org member could read org audit rows.
