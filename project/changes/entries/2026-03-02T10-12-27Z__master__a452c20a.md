# Project Change Entry

- Date/time (UTC): 2026-03-02T10:12:27Z
- Branch: master
- Base commit: a452c20a
  What changed:
- Updated `.github/workflows/retry-vercel-deploy.yml` to recover from missing deploy-hook secret by falling back to Vercel CLI deploy.
- Added bounded deploy retries (3 attempts with backoff) and increased workflow timeout from 10 to 30 minutes.
- Added GitHub secrets needed by the CLI fallback path: `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`.

Why:

- Production deploys are currently unstable and the existing retry workflow was hard-failing because `VERCEL_DEPLOY_HOOK_URL` was not configured.
- The CLI fallback keeps automated retries active without requiring a deploy hook and improves resilience during Vercel incident windows.

How to verify:

- Trigger workflow: `gh workflow run "Retry Vercel Deploy Until Synced" --ref master`.
- Check latest run logs include either `Triggering production deploy hook.` or `Falling back to Vercel CLI deploy with retries.`.
- Confirm workflow now progresses past secret validation and attempts deploy retries.

Open risks / TODO:

- Vercel platform-side `Deploying outputs` internal errors may still fail individual attempts until incident is fully resolved.
- No-cache deploy attempts can still OOM on Next.js build; cache-backed retries remain preferred.
