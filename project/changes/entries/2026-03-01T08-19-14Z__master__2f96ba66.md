# Project Change Entry

- Date/time (UTC): 2026-03-01T08:19:14Z
- Branch: master
- Base commit: 2f96ba66
  What changed:
- Migrated data fetching from client components to server components for dashboard widgets in both Organization and Individual applications.
- Refactored `DashboardClient` to accept `initialData` and pass it down to `DraggableDashboard` and individual widgets (`NextBestActionsWidget`, `MatchingReadinessCard`, `InterviewsFeedbackCard`, `MomentumMetricsCard`, `GoalsCard`, `ExploreCard`, `ProjectsCard`, `WhileAwayCard`, `RecentMatchesWidget`, `TopCandidatesWidget`, `PipelineSummaryWidget`, `ActionItemsWidget`, `InterviewsWidget`).
- Updated `indDataFetchers.ts` to directly use Supabase client for `getIndInterviewsData` to resolve Drizzle schema mismatches.
- Updated Server Component routes (`/app/o/[slug]/home/page.tsx` and `/app/i/home/page.tsx`) to pre-fetch widget data and pass it to their respective clients.

Why:

- To improve performance and maintainability by leveraging Next.js Server Components.
- To reduce client-side data fetching waterfalls and initial loading skeletons in Dashboards.

How to verify:

- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`
- Manual testing: loading the `/app/i/home` and `/app/o/[slug]/home` pages to verify widgets render immediately with server-fetched data.
