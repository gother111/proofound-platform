# Project Change Entry

- Date/time (UTC): 2026-03-01T22:53:00Z
- Branch: master
- Base commit: 83bc866c
  What changed:
- Hardened CV upload document identifiers in `src/components/expertise/cv-import/CvImportWizard.tsx`:
  - Added ASCII-safe opaque `document_id` generation.
  - Replaced filename-derived request IDs with request-time safe IDs.
  - Added local-to-request ID reconciliation so multipart, JSON fallback, OCR retry, and result merge all use consistent IDs.
- Hardened multipart parsing in `api/python/cv_import.py`:
  - Added robust UTF-8 decode error detection for direct, chained, and grouped exceptions.
  - Normalized multipart decode failures to a friendly validation message.
  - Added document ID sanitizer fallback for invalid or empty multipart IDs.
- Hardened proxy sanitization in `src/lib/expertise/python-cv-proxy.ts`:
  - Normalized codec error signatures in both JSON upstream payloads and plain-text upstream errors.
- Added route-level safety net in:
  - `src/app/api/expertise/cv-import/suggest/route.ts`
  - `src/app/api/expertise/cv-import/wizard-suggest/route.ts`
    to prevent raw codec text from leaking if upstream sanitization is bypassed.
- Updated and added tests:
  - `tests/ui/cv-import-wizard.test.tsx`
  - `tests/lib/python-cv-proxy.test.ts`
  - `tests/api/cv-import-suggest-route.test.ts`
  - `tests/api/cv-import-wizard-suggest-route.test.ts`
  - `tests/python/test_cv_import_multipart_errors.py`

Why:

- CV analyze upload occasionally failed with raw Python codec exceptions (`'utf-8' codec can't decode byte ...`) shown directly in the UI.
- The hotfix removes unsafe multipart metadata patterns and enforces consistent friendly error normalization across client, proxy, route, and Python parser layers.
