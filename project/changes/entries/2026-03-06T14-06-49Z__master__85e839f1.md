# Project Change Entry

- Date/time (UTC): 2026-03-06T14:06:49Z
- Branch: master
- Base commit: 85e839f1
  What changed:
- Made wizard analyze resilient to late-stage skill processing failures by catching skill-merge and Atlas verification errors per candidate in `src/app/api/expertise/cv-import/wizard-suggest/route.ts`.
- Added backward-compatible recovery metadata and candidate fallback reasons in `src/lib/expertise/cv-import-suggest.ts` and `src/lib/expertise/cv-import-wizard-types.ts`.
- Updated `src/components/expertise/cv-import/CvImportWizard.tsx` and `src/components/expertise/cv-import/AnalyzeProgressPanel.tsx` so partial recoveries show a warning state and real backend failures show specific messages instead of the generic analyze error.
- Added route and UI regression coverage for Atlas verification fallback, unreadable-document recovery, and progress-panel failure messaging.

Why:

- Production analyze failures were still able to collapse the whole wizard when late Atlas verification threw, even after extraction and entity parsing had succeeded.
- The previous UI masked useful error context with a generic failed-progress message, which made diagnosis and recovery harder for users.

How to verify:

- `npm run test -- tests/api/cv-import-wizard-suggest-route.test.ts tests/ui/cv-import-wizard.test.tsx`
- `npm run test -- tests/ui/cvjd-auto-suggest.test.tsx`
- `npm run typecheck`
- `npm run lint`
- Manual: upload a CV, trigger an Atlas-verification failure path, confirm the wizard still opens with a warning and usable partial results.

Open risks / TODO:

- Recovery telemetry writes to `analytics_events` only when service-role credentials are available; missing env prevents those inserts silently by design.
- This change degrades gracefully when Atlas verification fails, but it does not fix the underlying Atlas outage itself.
