# Project Change Entry

- Date/time (UTC): 2026-03-06T09:34:13Z
- Branch: master
- Base commit: 57436a82

What changed:

- Added a shared symbol-aware taxonomy normalizer for punctuation-heavy skills such as `C#`, `C++`, `Node.js`, `React.js`, `Next.js`, `CI/CD`, and `.NET`.
- Wired the new normalization contract into Atlas verification, manual skill suggestion scoring, and taxonomy alias tooling.
- Added a new DB migration to replace `public.normalize_skill_alias()`, repair affected alias rows, and prevent stale `NodeJS` / canonical-equivalent alias drift.
- Added direct regression tests for normalization semantics, Atlas verifier behavior, and CV-import mismatch scenarios.

Why:

- The old DB-side alias normalizer collapsed `C#` and `C++` to the same key and widened symbol-heavy skills into generic taxonomy matches.
- The app and tooling had drifted away from the DB normalization contract, which made matching less reliable and made alias-wave output inconsistent.

How to verify:

- `npm run test -- tests/lib/taxonomy-normalization.test.ts tests/lib/atlas-skill-verifier.test.ts tests/api/expertise-taxonomy-route.test.ts tests/lib/taxonomy-mismatch-regressions.test.ts tests/lib/cv-import-suggest-quality.test.ts`
- `npm run typecheck`
- `npm run lint`
- `npm run db:audit:migrations` and confirm the new migration is listed as unapplied until a safe local/dev DB target is available

Open risks / TODO:

- The new migration was not applied in this environment because only a remote Supabase pooler target is configured in `.env.local`.
- `npm run taxonomy:validate:aliases` still fails against the current remote data until the new normalization migration is applied.
