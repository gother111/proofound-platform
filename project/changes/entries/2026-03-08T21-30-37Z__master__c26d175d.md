What changed:

- Expanded canonical verification schema and contracts for Block 7 with richer verification kinds, lifecycle states, verifier classes, verification slots, dispute state, contradiction/dispute tables, and lifecycle timestamps.
- Added `src/lib/verification/policy.ts` to summarize canonical verification records into conservative trust labels, compatibility fields, freshness handling, and active issue states.
- Updated verification status APIs, public portfolio trust cues, org trust cues, verification gates, and work-email workflow transitions to consume canonical policy summaries.
- Added the Block 7 policy doc at `docs/verification-policy-mvp.md`, registered it in `docs/DOCS_REGISTRY.md`, and added focused verification policy tests.

Why:

- Proofound needs one auditable verification policy where badges mean something narrow, stable, and explainable instead of relying on scattered legacy booleans.
- Contradictions, expiry, disputes, and downgrade paths must be explicit in the data model so trust can degrade safely without deleting history.
