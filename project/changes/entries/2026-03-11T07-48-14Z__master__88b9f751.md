# Project Change Entry

- Date/time (UTC): 2026-03-11T07:48:14Z
- Branch: master
- Base commit: 88b9f751

What changed:

- Added a standalone `apps/api` Next app that mirrors the existing `src/app/api` surface through generated thin wrappers.
- Added `scripts/generate-api-app-wrappers.mjs` and root scripts to sync, typecheck, and build the extracted API app.
- Added an `apps/api/api/python/cv_import.py` wrapper and `apps/api/vercel.json` so the API app can own cron routes and the Python CV import function when its Vercel project is created.

Why:

- The API surface is the largest single part of the monolith build, so extracting it first is the highest-value architectural split from the approved plan.
- Reusing the existing route handlers by reference keeps the external API contract unchanged while avoiding a hand migration of 311 endpoints.

How to verify:

- Run `npm run api:wrappers:sync` and confirm `apps/api/app/api` contains 311 generated route wrappers.
- Run `npm run typecheck:api`.
- Run `npm run build:api`.
- Run `npm run lint` and `npm run typecheck` for the root app to confirm the extraction scaffolding did not regress the current deployment path.

Open risks / TODO:

- Same-domain routing to a dedicated API Vercel project is not wired yet. This commit prepares the standalone app but does not cut traffic over.
- Some local build output still prints repeated `DATABASE_URL is missing` warnings because API route imports execute env checks during static analysis without production secrets.
- Product and marketing app extraction remain pending.
