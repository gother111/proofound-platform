# Project Change Entry

- Date/time (UTC): 2026-03-09T14:35:28Z
- Branch: master
- Base commit: 9b8f4964

What changed:

- Made the shared mobile bottom nav shell in `src/components/app/LeftNav.tsx` non-hit-testable outside the actual nav items, while keeping the links and locked buttons interactive.
- Made the global cookie banner wrapper in `src/components/CookieBanner.tsx` non-hit-testable outside the consent card so lower landing-page controls are not blocked by the fixed container.
- Added focused Playwright coverage for landing CTA clickability with the cookie banner visible and for mobile individual/org profile actions above the bottom nav.

Why:

- Visible buttons on profile pages were being covered by fixed shell overlays. The mobile nav was intercepting taps in its empty gutter space.
- The landing page also had a fixed bottom wrapper pattern in the cookie banner that could block clicks outside the actual banner card.

How to verify:

- Run `npm run typecheck`.
- Run `npm run lint`.
- Run `npm run build`.
- Run `PLAYWRIGHT=true npm run test:e2e:landing`.
- Run `PLAYWRIGHT=true npm run test:e2e:landing:visual`.
- Run `NEXT_PUBLIC_USE_MOCK_SUPABASE=true PLAYWRIGHT=true node ./scripts/playwright-node20.mjs test e2e/mobile-smartphone.spec.ts --project=chromium --reporter=line --workers=1 --grep "individual profile action buttons remain clickable"`.
- Run `NEXT_PUBLIC_USE_MOCK_SUPABASE=true MOCK_ORG_MODE=true PLAYWRIGHT=true node ./scripts/playwright-node20.mjs test e2e/mobile-smartphone.spec.ts --project=chromium --reporter=line --workers=1 --grep "organization profile actions remain clickable"`.

Open risks / TODO:
