# Project Change Entry

- Date/time (UTC): 2026-03-10T17:50:53Z
- Branch: master
- Base commit: 0a22dfb7
  What changed:
- Completed the proof-first canonicalization so Proof Pack is the main reviewable proof object and Artifact remains the atomic evidence unit across profile trust, matching, org review, public portfolio, export, and portability.
- Extended canonical proof schema and added backfill migrations for orphaned legacy `skill_proofs` into `proof_artifacts`, `proof_packs`, and `proof_pack_items`.
- Added canonical pack aggregation and skill-proof summary helpers, then switched trust snapshots, readiness/profile completion, expertise stats, dashboard/profile mapping, public portfolio proof projection, verify token previews, and user export/import to canonical proof state.
- Updated compatibility proof create/delete flows and onboarding write-through so legacy proof entry points keep canonical proof state in sync.
- Added proof-first coverage for canonical pack rollups, public-safe visibility boundaries, expertise stats, verification preview, proof route compatibility, export/import round-trip, freshness transitions, and privacy-safe exports.

Why:

- The MVP PRD and technical requirements require Proof Pack to be the canonical proof-first core. Loose evidence counts and direct `skill_proofs` reads were still competing with that model in several product surfaces.
- This change centralizes proof freshness, verification rollups, visibility narrowing, and portability boundaries so public-safe and owner-full proof projections come from one canonical source.
