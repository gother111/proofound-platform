Why:

- Repeated Proofound sessions were spending time on the same post-ticket admin loop: identify the active `PRO-*` issue, inspect git state, determine missing verification, and draft the Linear-ready closeout update.
- The helper makes that workflow deterministic for both humans and the recurring Codex automation without mutating git state or final Linear status.

How to verify:

- `npm run test -- tests/scripts/proofound-ticket-finisher.test.ts`
- `node scripts/proofound-ticket-finisher.mjs`
- `npm run lint`
- `npm run typecheck`
- `npm run docs:freshness`

Open risks / TODO:

- The helper intentionally does not post final Linear completion comments or move issues to `In Review`; that remains a human review step.
