What changed:

- Added an explicit guided-flow escape hatch in `src/components/profile/GuidedProfileSetupView.tsx`:

  - New `Open full profile` secondary CTA in the guided header.

- Extended `tests/ui/editable-profile-purpose-gating.test.tsx`:
  - Added override case for incomplete profiles with `profileView=full`.
  - Added interaction test for guided CTA preserving query params when switching to full profile.
  - Updated `next/navigation` and guided view mocks for `replace`, `usePathname`, and new CTA callback.

Why:

- PRO-73 investigation showed no active blocker in completion computation, but users in guided mode needed an explicit path to full profile without changing the default PRO-64 behavior.
- Product direction selected hybrid UX: default guided flow + explicit full-profile opt-in.
