# Project Change Entry

- Date/time (UTC): 2026-02-28T18:37:24Z
- Branch: master
- Base commit: b8762d88
  What changed:
- Hardened skill loading in `GET /api/expertise/verifications/custom/artifacts` with a two-step strategy:
  - Primary: existing relation select via `skills_taxonomy!skills_skill_code_fkey`.
  - Fallback: select skills without relation + load taxonomy by `skill_code` and stitch names in memory.
- Hardened skill validation/loading in `POST /api/expertise/verifications/custom/request` with the same two-step strategy.
- Added API regression tests for fallback behavior in `tests/api/custom-verification-routes.test.ts`:
  - artifacts route returns skills when relation alias lookup fails and fallback succeeds.
  - custom request POST succeeds when selected-skill relation alias lookup fails and fallback succeeds.

Why:

- In environments with FK/relationship alias drift, hardcoded PostgREST relation aliases can fail.
- This caused missing skill artifacts in Custom verification request (and could also break selected-skill validation on submit).
- The fix preserves API contracts while making skill paths resilient to alias metadata mismatch.
