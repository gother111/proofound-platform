  What changed:
- Broadened the CV wizard client fallback ladder in `src/components/expertise/cv-import/CvImportWizard.tsx` so initial multipart extract failures now retry through local PDF text extraction for proxy unavailable, proxy timeout, proxy invalid contract, multipart metadata errors, and generic `Failed to extract CV text` 5xx failures.
- Added `Failed to extract CV text` to the generic wrapper error set so the UI prefers the backend message instead of surfacing the bare extract label when the request truly cannot recover.
- Added UI regression tests for invalid-contract extract failure recovery, generic extract 500 recovery, and parser-failure messaging during local fallback.

Why:

- The Analyze button could stop at the initial Python `/extract` stage and show `Failed to extract CV text`, even though the client already had enough information to recover via local PDF extraction.
- The user-facing error text was too generic and hid the actual cause when recovery failed.

How to verify:

- `npm run typecheck`
- `npm run lint`
- Manual: upload a CV, force a failing multipart `/extract` response, and confirm the wizard retries via local extraction instead of stopping at the generic extract error.
