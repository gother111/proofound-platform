# Project Change Entry

- Date/time (UTC): 2026-04-02T13:16:19Z

What changed:

- Added a new root `DESIGN.md` that consolidates Proofound's visual principles, token usage, surface guidance, and agent-facing UI rules.
- Updated `AGENTS.md` so UI, landing, public-page, and visual-system work explicitly reads `DESIGN.md` before editing.

Why:

- The repo had design truth spread across tokens, Tailwind config, MVP docs, and existing components, but no single UI contract for agents to follow.
- This applies the `awesome-design-md` pattern in a Proofound-native way without importing a third-party visual identity.

How to verify:

- Read `DESIGN.md` and confirm it matches the existing brand token files, motion tokens, and MVP scope language.
- Confirm `AGENTS.md` now points relevant UI work toward `DESIGN.md`.
- Run `npm run docs:freshness`, `npm run lint`, and `npm run typecheck`.

Open risks / TODO:

- `DESIGN.md` is a first-pass contract and may need refinement as more authenticated product surfaces are standardized.
- `npm run docs:freshness` still reports pre-existing repo warnings unrelated to this change.
