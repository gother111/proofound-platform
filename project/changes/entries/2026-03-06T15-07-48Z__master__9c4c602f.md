# Project Change Entry

- Date/time (UTC): 2026-03-06T15:07:48Z
- Branch: master
- Base commit: 9c4c602f

What changed:

- Moved the daily scheduler split to the intended model: `vercel.json` now owns `decision-reminders`, `refresh-matches`, `refresh-matches-worker`, and `sla-enforcement`.
- Removed fairness-note generation from the legacy `account-deletion-workflow` cron route so it is now a compatibility no-op.
- Reconciled cron-job.org ownership through `scripts/sync-cron-job-org.mjs`, including disabling external `sla-enforcement` and guarding the retired overlap jobs.
- Raised the default `MATCHING_REFRESH_WORKER_BATCH_SIZE` to `100` and updated the cron docs and env defaults to match.

Why:

- The live system had duplicate fairness-note generation and split ownership drift between Vercel Cron and cron-job.org.
- The match refresh worker default batch size was too small to reliably clear one day of queued work in a single daily run.
