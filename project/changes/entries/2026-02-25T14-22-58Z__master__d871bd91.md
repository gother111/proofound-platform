# Project Change Entry

- Date/time (UTC): 2026-02-25T14:22:58Z

- Base commit: d871bd91
  What changed:
- Implemented PRO-37 follow-up for skill proofs and add-flow verification.
- Added document proof support end-to-end:
- `src/db/schema.ts` now includes `document` in `skill_proofs.proof_type`.
- New migration `src/db/migrations/20260225170000_add_document_skill_proof_type.sql` updates `skill_proofs` proof type check constraint.
- `POST /api/expertise/user-skills/[id]/proofs` now accepts `proofType: "document"` and optional `filePath`, with validation requiring at least one of title/url/filePath.
- `DELETE /api/expertise/user-skills/[id]/proofs/[proofId]` now performs best-effort cleanup of uploaded files from `user-uploads`.
- Updated edit-skill proof UI to support document upload via `uploadFile` + `validateFile`.
- Updated add-skill step 4 to support URL vs document proof sources and inline verification-request fields.
- Add-skill save flow now supports partial-success messaging:
- skill created + proof attached + verification requested (all success), or explicit warnings when proof/verification fails after skill creation.
- Added/updated tests:
- `tests/api/expertise-user-skill-proofs-route.test.ts`
- `tests/ui/edit-skill-window-proofs.test.tsx`
- `tests/ui/add-skill-drawer-proof-verification.test.tsx`

Why:

- Latest PRO-37 feedback reported two remaining UX gaps after earlier fixes:
- add-skill proof support was URL-only (no document upload).
- verification request was not available during skill creation.
- The implemented changes close both gaps while preserving non-blocking behavior for skill creation.
