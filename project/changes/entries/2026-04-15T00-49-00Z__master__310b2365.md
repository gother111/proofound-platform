What changed:

- Disabled weekly digest eligibility for all users at the data layer by backfilling `notification_preferences` and changing the DB/schema defaults to off.

- Added `/api/cron/weekly-digest` to the cron-job.org legacy disable set so repo-managed cron reconciliation can turn off any stray external digest job.

Why:

- Users were still receiving Monday weekly digest emails even though the application code path was documented as disabled.
- Production had active users with no stored notification preference rows, which made older eligibility logic treat them as opted in by default.
