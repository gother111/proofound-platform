# Project Change Entry

- Date/time (UTC): 2026-03-09T13:02:23Z
- Branch: master

  What changed:
- Added Proofound-native GEO tooling and surfaces:
- `scripts/geo-audit.mjs` plus `npm run geo:audit` to audit public marketing, legal, `robots.txt`, `sitemap.xml`, `llms.txt`, and `llms-full.txt`.
- `src/app/llms.txt/route.ts` and `src/app/llms-full.txt/route.ts` backed by `src/lib/seo/llms.ts`.
- Reusable SSR JSON-LD helpers in `src/lib/seo/json-ld.ts` and `src/components/seo/JsonLdScripts.tsx`.
- Expanded SSR JSON-LD coverage on the homepage, audited marketing pages, legal pages, cookie settings, and public portfolio pages.
- Added focused tests for JSON-LD helpers and `llms` routes.

Why:

- Proofound already had canonical metadata and privacy-aware sitemap logic, but lacked `llms.txt` support, page-level structured data coverage, and a repo-native GEO audit workflow.
- The goal was to reuse external GEO repos as reference patterns without importing their Claude-specific runtime structure into Proofound.

How to verify:

- `npm run typecheck`
- `npm run lint`
- `npx vitest run tests/lib/public-profile-metadata.test.ts tests/lib/public-json-ld.test.ts tests/app/llms-routes.test.ts tests/ui/public-portfolio-page.test.tsx tests/ui/public-org-portfolio-page.test.tsx`
- `npm run geo:audit -- https://proofound.io`
- `PORT=3005 npm run dev` then `npm run geo:audit -- http://127.0.0.1:3005`

Open risks / TODO:

- The audit script is intentionally heuristic and does not replace Lighthouse, Rich Results testing, or live AI-engine ranking measurements.
- Production `proofound.io` still showed missing `llms.txt` and non-homepage JSON-LD during verification because the new changes are local only until deployment.
