  What changed:
- Hardened JSON-LD serialization in `JsonLdScripts` by replacing raw `JSON.stringify` output with an escaped serializer that encodes `<` as `\\u003c` before injecting into `application/ld+json` scripts.
- Added a regression test that proves payloads containing `</script><script>...` are escaped and cannot break out of the JSON-LD script tag.

Why:

- Prevent stored XSS on public portfolio pages where user-controlled values are rendered inside JSON-LD script tags.

How to verify:

- `npm run test -- tests/lib/public-json-ld.test.ts`
- `npm run lint`
- `npm run typecheck`
