# Project Change Entry

- Date/time (UTC): 2026-03-10T20:33:12Z
- Branch: master
- Base commit: 5a2984be

What changed:

- Wired `.next/cache` cleanup into both `prebuild` and `postbuild` in `package.json`.
- Updated `scripts/postbuild-cleanup.mjs` so cache deletion runs automatically for Vercel builds, while remaining a no-op for normal local workflows unless `NEXT_CLEAN_BUILD_CACHE=1` is set.

Why:

- Vercel production builds were restoring and extending multi-GB Next.js webpack cache packs, then getting OOM-killed during `next build`.

How to verify:

- Run `VERCEL=1 npm run build` and confirm the build completes.
- Confirm `.next/cache` is absent after the build finishes.
- Run `npm run lint`, `npm run typecheck`, and `npm run docs:freshness`.

Open risks / TODO:

- The fix targets runaway webpack cache reuse. If OOM returns with a clean cache, the next step is to profile specific heavy server imports and disable filesystem cache more aggressively for Vercel.
